## 3. Explain Mode Summary

Attack Explanation:
The attacker positions a relay between the client and bank gateway, then tries to read the transaction payload while it is moving through the network.

Defense Explanation:
AES-256-GCM encrypts the transaction before transit, so the attacker can still capture traffic but cannot understand or modify the protected payload.

Attack Flow:
- Client prepares a bank transaction request.
- Attacker relay intercepts the request route.
- Plaintext mode exposes account, merchant, amount, and nonce details.
- Captured data can be copied into attacker logs.

Defense Flow:
- AES-256-GCM creates a random nonce and encrypts the payload.
- Only ciphertext is visible during interception.
- Authentication tag protects against payload tampering.
- Gateway receives a verified encrypted request.

Key Takeaway:
Encryption does not always stop interception, but it prevents the attacker from reading useful information.
